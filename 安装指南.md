## 🚀 快速开始

### 第 1 步：安装 Node.js

1. 访问 [Node.js 官网](https://nodejs.org/)
2. 下载并安装 **LTS 版本**（推荐 v18 或更高）
3. 安装完成后验证：
   ```bash
   node -v
   npm -v
   ```

### 第 2 步：安装 MongoDB

#### 方式 A：ZIP 解压版（推荐）

1. **下载 MongoDB**
   - 访问：https://www.mongodb.com/try/download/community
   - Version: **7.0.x** 或 **8.0.x**
   - Platform: **Windows**
   - Package: **zip**
   - 点击下载

2. **解压到 F 盘**
   ```
   F:\MongoDB\
   ```
   确保解压后结构如下：
   ```
   F:\MongoDB\
   ├── bin\
   │   ├── mongod.exe
   │   ├── mongod.cfg
   │   └── mongosh.exe (可选)
   ├── Data\
   └── Log\
   ```

3. **创建必要目录**
   ```bash
   mkdir F:\MongoDB\Data
   mkdir F:\MongoDB\Log
   ```

4. **创建配置文件**
   
   创建文件：`F:\MongoDB\bin\mongod.cfg`
   
   内容：
   ```yaml
   systemLog:
     destination: file
     path: F:\MongoDB\Log\mongod.log
     logAppend: true
   storage:
     dbPath: F:\MongoDB\Data
   net:
     bindIp: 127.0.0.1
     port: 27017
   ```

5. **启动 MongoDB**
   
   以**管理员身份**打开 CMD，执行：
   ```cmd
   F:\MongoDB\bin\mongod.exe --config "F:\MongoDB\bin\mongod.cfg" --install
   net start MongoDB
   ```

#### 方式 B：MSI 安装包

1. 下载 MSI 安装包并运行
2. 选择 "Complete" 安装
3. 勾选 "Install MongoDB as a Service"
4. 完成安装后，服务会自动启动

## 👤 创建管理员账号

由于登录页面没有注册入口，需要通过浏览器控制台创建：

### 步骤：

1. **打开浏览器开发者工具**
   - 访问 http://localhost:5173
   - 按 `F12` 键

2. **切换到 Console 标签**

3. **粘贴并执行以下代码**：

```javascript
fetch('/api/auth/register', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    username: 'admin',
    password: 'Admin@123',
    email: 'admin@example.com'
  })
})
.then(response => response.json())
.then(data => {
  console.log('✅ 注册成功!');
  console.log('用户名：admin');
  console.log('密码：Admin@123');
})
.catch(error => console.error('❌ 注册失败:', error));
```

4. **看到成功提示后**，关闭开发者工具

5. **使用以下账号登录**：
   - 用户名：`admin`
   - 密码：`Admin@123`

## 🔐 可选：启用 MongoDB 认证（生产环境推荐）

### 为什么要启用认证？

默认情况下 MongoDB 不需要密码即可访问，适合开发环境。生产环境必须启用认证。

### 启用认证步骤

#### 1️⃣ 安装 MongoDB Shell（如果未安装）

1. 下载：https://www.mongodb.com/try/download/shell
2. 解压并将 `mongosh.exe` 复制到：`F:\MongoDB\bin\`
3. 验证：`F:\MongoDB\bin\mongosh.exe --version`

#### 2️⃣ 创建管理员用户

```bash
F:\MongoDB\bin\mongosh.exe
```

在 MongoDB Shell 中执行：

```javascript
// 切换到 admin 数据库
use admin

// 创建超级管理员
db.createUser({
  user: "root",
  pwd: "YourStrongPassword123!",
  roles: ["root"]
})

// 切换到项目数据库
use permission_manager

// 创建应用专用用户
db.createUser({
  user: "perm_app",
  pwd: "AppPassword456!",
  roles: [{role: "readWrite", db: "permission_manager"}]
})

// 退出
exit
```

#### 3️⃣ 修改 MongoDB 配置

编辑文件：`F:\MongoDB\bin\mongod.cfg`

在文件末尾添加：
```yaml
security:
  authorization: enabled
```

保存文件。

#### 4️⃣ 重启 MongoDB

```cmd
net stop MongoDB
net start MongoDB
```

#### 5️⃣ 更新项目配置

编辑 `F:\AI Project\backend\.env`：

```env
MONGODB_URI=mongodb://perm_app:AppPassword456!@localhost:27017/permission_manager?authSource=permission_manager
```

#### 6️⃣ 重启后端服务

```bash
cd F:\AI Project\backend
npm run dev
```

## 🛠️ 常见问题

### Q1: MongoDB 服务无法启动

**解决方案**：
1. 检查数据目录是否存在：`F:\MongoDB\Data\`
2. 检查日志目录是否存在：`F:\MongoDB\Log\`
3. 查看日志文件：`F:\MongoDB\Log\mongod.log`
4. 确保 27017 端口未被占用

### Q2: npm install 失败

**解决方案**：
```bash
# 使用淘宝镜像加速
npm config set registry https://registry.npmmirror.com
npm install
```

### Q3: 端口被占用

**后端 3000 端口被占**：
- 编辑 `backend\.env`，修改 `PORT=3000` 为其他端口

**前端 5173 端口被占**：
- 编辑 `frontend\vite.config.js`，修改 `port: 5173` 为其他端口

### Q4: 无法访问 http://localhost:5173

**解决方案**：
1. 确认前端服务已启动
2. 检查防火墙设置
3. 清除浏览器缓存后重试

### Q5: 接口请求失败（401 错误）

**解决方案**：
1. 确认已登录（有 Token）
2. 检查 Token 是否过期
3. 重新登录后再试
